## What Lives Here

**SIMULATIONS.md** — 10 edge-case patterns with proven moves

Pull when:
- Stuck loop (same tool 3+ times, same result)
- New project (CONTEXT.md empty or stub)
- Dangerous command (irreversible data loss risk)
- Vague task ("make it better", "fix auth")
- 10-file change (5+ files in parallel)
- Review request ("check this", PR review)
- Architecture decision ("how should I structure", A or B)
- Memory moment (significant decision made)
- Pushback (Level 2+ triggered)
- First session on project

---

## How to Use

Not pre-loaded. Pull specific files when situation calls for it.

"Feels like stuck loop" → read SIM-01  
"User asked for review" → read SIM-06  
"Big refactor across 10 files" → read SIM-05  
"First session, new project" → read SIM-10

Simulations are pattern matches, not rules. When situation rhymes — right move is already known.
