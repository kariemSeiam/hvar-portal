# Context signals

When to suggest doc updates. Declarative.

---

## Signals that trigger update suggestions

| Signal | Example phrases |
|--------|-----------------|
| Decision | "We decided to...", "Let's use...", "I think we should..." |
| Idea | "What if...", "Maybe we could...", "We could add..." |
| Planning | "Let's do X first", "Phase 1 should...", "We need to..." |
| Design | "The layout should...", "I want it to look like...", "The UI needs..." |
| Question resolved | Question asked and answered; uncertainty clarified |
| Change | "Actually, let's change...", "Instead of X, do Y" |

---

**When to suggest:** Decision agreed · Idea worth recording · Plan/roadmap changed · Design choice · Question answered · Direction pivoted.

---

## File mapping (idea folders)

| Signal | Suggested files |
|--------|-----------------|
| Decision | discussion.md, planning.md |
| New idea | thinking.md, uiux.md |
| Task added | planning.md |
| Design | uiux.md, discussion.md |

Canonical project docs: **docs/** (see docs/INDEX.md). Idea folders in dev/ use thinking, planning, uiux, discussion, notes.
