📋 Service Cycle Steps

1 Customer Call - Order Not Received

Customer calls complaining about not receiving their order. Agent confirms customer data and order details.

POST /api/customer-service/tickets
Customer: +201234567890
Ticket Type: Order Issue

2 Order Delivery Confirmation

Order is successfully delivered to customer. Customer confirms receipt.

PUT /api/orders/{order_id}/status
Status: delivered

3 Customer Calls for Maintenance

Customer calls requesting maintenance for their product. Agent explains 200 EGP shipping fee.

POST /api/customer-service/tickets
Ticket Type: Maintenance Request
Fee: 200 EGP

4 Create Pickup Order

Create pickup order to collect the product from customer for maintenance.

POST /api/unified-service/service-actions
Action Type: pickup
COD: 200 EGP

5 Hub Scanning & Inspection

Product arrives at hub. Agent scans and inspects the product condition.

POST /api/service-actions/hub/scan
Tracking:

6 Create Maintenance Cycle

Create maintenance cycle with technician assignment and parts allocation.

POST /api/maintenance/cycles
Cycle Type: corrective
Priority: medium

7 Complete Maintenance

Technician completes maintenance. Quality check passed.

POST /api/maintenance/cycles/{cycle_id}/complete
Quality Score: 9/10

8 Create Return Order

Create return order with COD -200 EGP (refund of shipping fee).

POST /api/unified-service/service-actions
Action Type: return
COD: -200 EGP

9 Final Customer Call

Call customer to confirm return order details before delivery.

POST /api/customer-service/tickets
Ticket Type: Return Confirmation

10 Final Delivery

Return order delivered to customer. Cycle completed successfully.

PUT /api/orders/{return_order_id}/status
Status: delivered
